# Load Test Results Directory

This directory contains all load test execution results and analysis reports.

## File Organization

```
results/
├── scenario_1_baseline_YYYY-MM-DDTHH-mm-ss-sssZ.json
├── scenario_2_disruption_peak_YYYY-MM-DDTHH-mm-ss-sssZ.json
├── scenario_3_stress_breakpoint_YYYY-MM-DDTHH-mm-ss-sssZ.json
├── stress_breakpoint_analysis_YYYY-MM-DDTHH-mm-ss-sssZ.md
├── scenario_4_fraud_abuse_YYYY-MM-DDTHH-mm-ss-sssZ.json
├── fraud_security_report_YYYY-MM-DDTHH-mm-ss-sssZ.md
└── performance_trends.md (manual analysis)
```

## Result Types

### JSON Files
Complete k6 test execution data including:
- Request/response metrics
- Custom counter values
- Threshold pass/fail status
- Timing breakdown by endpoint

### Markdown Reports
Human-readable analysis for:
- Stress breakpoint identification
- Security testing effectiveness
- Performance trend analysis
- Capacity planning recommendations

## Retention Policy

- Keep last 30 days of test results
- Archive monthly summaries for long-term trends
- Critical performance regressions: permanent retention

---

Results are automatically generated by k6 `handleSummary()` functions in each scenario script.
